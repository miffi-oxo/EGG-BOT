### `utils/tts/` 폴더가 필요한 이유

TTS 커스텀 사운드(예: `[기분좋은향기]`, `[오셨군요]` 같은 효과음)의 실제 `.mp3` 파일이 저장되는 곳입니다. `handler/tts.py`의 `AUDIO_BASE_PATH`가 이 경로를 가리킵니다.

- `utils/tts/*.mp3` — 모든 서버에서 공통으로 쓰는 기본 효과음 (`mokoko.mp3`, `welcome.mp3`, `hello.mp3`, `escape.mp3`)
- `utils/tts/<guild_id>/*.mp3` — `/tts커스텀` 명령어로 각 서버가 개별 등록한 커스텀 사운드. 서버별 인덱스 파일(`custom_sounds.json`)도 이 하위 폴더에 함께 생성됩니다.

코드가 아니라 **오디오 데이터가 실제로 들어있는 폴더**라서, 다른 유틸 스크립트처럼 지워도 되는 게 아니라 반드시 있어야 하는 리소스 폴더입니다.

> **폴더 자체는 미리 안 만드셔도 됩니다.** `utils/tts/`, `utils/tts/<guild_id>/` 둘 다 필요한 시점(예: `/tts커스텀`으로 서버가 처음 사운드를 추가할 때)에 코드가 `os.makedirs(..., exist_ok=True)`로 자동 생성합니다.
>
> 다만 **기본 효과음 4개(`mokoko.mp3`, `welcome.mp3`, `hello.mp3`, `escape.mp3`)는 자동으로 생기지 않습니다.** 이건 데이터가 아니라 실제 음원 파일이라 코드가 만들어낼 수 없으므로, `utils/tts/` 경로에 직접 업로드해둬야 `[기분좋은향기]` 등 기본 태그가 정상 재생됩니다. 서버별 커스텀 사운드는 `/tts커스텀` 명령어로 봇 안에서 바로 추가되므로 별도 업로드가 필요 없습니다.

### `data/` 폴더 — 직접 안 만드셔도 됩니다

아래 파일들은 최초 실행 시 코드가 **자동으로 생성**합니다. `data/` 폴더 자체도 필요할 때 자동으로 만들어져서, 미리 준비할 필요가 없습니다.

| 파일 | 만드는 시점 | 담당 |
|---|---|---|
| `tts_channels.json` | `/tts` 명령어로 채널을 처음 지정할 때 | `core/tts_channels.py` |
| `tts_engines.json` | `/엔진` 명령어로 목소리를 처음 지정할 때 | `core/tts_engine_manager.py` |
| `raid_gold.json` | 봇 최초 구동 시 (하드코딩된 기본 레이드 데이터로 자동 생성) | `core/raid_data.py` |
| `raid_channel.json` | `/레이드채널` 명령어로 포럼 채널을 처음 지정할 때 | `core/raid_channel.py` |
| `raids.json` | `/레이드` 명령어로 게시물을 처음 생성할 때 | `cogs/raid_schedule.py` |

봇을 재구동해도 이 파일들에 저장된 값은 그대로 유지됩니다.

## ⚙️ 실행 방법

1. `requirements.txt`의 의존성을 설치합니다.
2. 프로젝트 루트에 `.env` 파일을 만들고 아래 내용을 채웁니다. (디스호스트를 쓰신다면 "간편 설정 → 환경변수(.env) 설정" 카드로 등록해도 동일하게 동작합니다.)

   ```env
   BOT_TOKEN=
   EGG_ID=
   LOSTARK_API_KEY=
   ```

   | 변수 | 설명 |
   |---|---|
   | `BOT_TOKEN` | 디스코드 개발자 포털에서 발급받은 봇 토큰 |
   | `EGG_ID` | 봇 소유자(계란) 디스코드 사용자 ID. `/메뉴추가`, `/클골추가` 등 "계란 전용" 명령어의 권한 체크에 사용됩니다. 비워두면 계란 전용 명령어는 아무도 사용할 수 없습니다. |
   | `LOSTARK_API_KEY` | 로스트아크 공식 오픈 API 키. 레이드 참가신청 시 캐릭터 레벨/직업 조회에 사용됩니다 ([발급 페이지](https://developer-lostark.game.onstove.com)). 비워두면 참가신청/닉네임변경이 항상 실패합니다. |

   디스코드 사용자 ID는 디스코드 앱에서 **설정 → 고급 → 개발자 모드**를 켠 뒤, 본인 프로필을 우클릭 → **ID 복사**로 확인할 수 있습니다.

3. `python bot.py`로 실행합니다.

```
discord.py==2.7.1
PyNaCl>=1.5.0
davey
httpx>=0.27.0
aiohttp>=3.9.0
navertts
edge-tts>=6.1.0
imageio-ffmpeg>=0.5.0
```

> `davey`는 Discord의 음성 E2EE(DAVE) 프로토콜 지원에 필요하고, `imageio-ffmpeg`는 시스템에 ffmpeg가 없어도 정적 바이너리를 자체적으로 받아와 TTS 오디오 변환에 사용됩니다. `httpx`는 TTS의 Lavalink 연동뿐 아니라 레이드 참가신청 시 로스트아크 API 호출(`core/lostark_api.py`)에도 사용됩니다.

|---|---|---|

| `/클골추가 <레이드> <난이도> <골드> <귀속골드> [입장레벨] [딜러정원] [서포터정원]` | 새 조합 추가 (총합 자동 계산, 정원 기본값 딜러3/서포터1) | 계란 전용 |
| `/클골수정 <레이드> <난이도> <골드> <귀속골드> [입장레벨] [딜러정원] [서포터정원]` | 기존 조합 수정 | 계란 전용 |

| 명령어 | 설명 | 권한 |

### 레이드 일정
| 명령어 | 설명 | 권한 |
|---|---|---|
| `/레이드채널 <채널>` | 레이드 모집 게시물을 올릴 포럼 채널 지정 (서버당 1개, 이미 지정돼 있으면 덮어씀) | 관리자 |
| `/레이드채널해제` | 레이드 채널 지정 해제 | 관리자 |
| `/레이드채널정보` | 현재 지정된 레이드 채널 확인 (채널이 삭제된 경우 자동으로 지정 해제) | 전체 |
| `/레이드 <제목>` | 날짜/시/분/레이드/난이도 모달 → 제출 후 "설명 작성하기" / "설명 없이 게시" 선택 → 지정된 포럼 채널에 게시물 생성 | 전체 |

**레이드 선택지**: `/클골`에 등록된 조합 외에 **"기타"**를 고를 수 있어요. "기타"는 난이도 선택이 필요 없고(더미 옵션이 기본 선택돼 있음), 정원은 16인(딜러12/서포터4)으로 고정되며 입장레벨 제한이 없어요. 레이드가 아닌 다른 컨텐츠 모집용이라 `/클골`에는 나오지 않습니다.

**시간 검증**: 이미 지난 날짜/시간이나 등록되지 않은 (레이드, 난이도) 조합은 제출이 막히고, 입력했던 값이 그대로 채워진 "다시 입력" 버튼이 떠요.

게시물에 달리는 버튼:
| 버튼 | 설명 | 권한 |
|---|---|---|
| 참가신청 | 캐릭터 닉네임 입력 → 로스트아크 API로 레벨/직업/전투력 조회 → 존재하지 않는 캐릭터거나 입장레벨 미달이면 거부 → 딜러/서포터 겸용 직업(도화가·바드·홀리나이트·발키리)만 역할 선택창이 뜨고 나머지는 자동 딜러 → 자리 있으면 즉시 등록, 없으면 대기열 등록. 한 사람당 캐릭터 1개만 신청 가능 | 전체 |
| 참가변경 | 이미 신청한 캐릭터의 닉네임만 변경 (역할·대기 순서는 유지) | 전체 |
| 참가취소 | 본인 신청 취소. 참가자였다면 대기열에서 같은 역할 1순위를 자동 승격 | 전체 |
| 대기열 명단 | 현재 대기열 확인 | 전체 |
| ⚙️ 관리 | 아래 참고 | 서버 관리자 또는 작성자 |

**⚙️ 관리 패널 세부 버튼**
| 버튼 | 설명 |
|---|---|
| 일정수정 | 날짜/시/분(+설명)만 재입력 (레이드/난이도/정원은 안 바뀜) |
| 제목수정 | 게시물 제목의 자유 텍스트 부분만 수정 |
| 삭제 | "정말로 삭제하시겠습니까?" 확인 후 실제 삭제 |
| 강제참여 | 서버 멤버 검색 또는 "용병"(디스코드 계정 없는 외부인) 추가. 캐릭터는 실제 조회를 거치며, 존재하지 않는 캐릭터면 강제참여라도 거부됨. 정원·중복 신청 체크도 동일하게 적용 |
| 강제변경 | 기존 참가자/대기열 중 하나를 골라 관리자가 대신 캐릭터를 변경 |
| 강제취소 | 기존 참가자/대기열 중 하나를 골라 관리자가 대신 취소 |

> 레이드 시작 10분 전에는 참가자 전원을 태그하는 알림이 게시물(스레드)에 자동으로 올라옵니다.
> 레이드 시작 30분 후에는 게시물이 삭제되지 않고 **포스트 닫기 + 포스트 잠그기**(archived + locked) 상태로 자동 마감됩니다 (기록은 남되 더 이상 조작 불가).
> 두 기능 모두 1분마다 확인하는 백그라운드 작업(`cleanup_loop`)으로 동작하며, 일정수정으로 시간을 바꾸면 알림 발송 여부가 초기화돼 새 시간 기준으로 다시 알림이 갑니다.

## 🗒️ 참고

- `계란 전용` 명령어는 `.env`의 `EGG_ID`(봇 소유자 디스코드 ID)와 명령어 실행자 ID를 비교하는 방식으로 권한을 체크합니다 (`core/config.py`에서 로드, `cogs/cleargold.py`·`cogs/menu.py`에서 사용).
- `관리자` 권한 명령어는 해당 서버의 관리자(Administrator) 권한을 가진 사용자만 사용할 수 있습니다.
